# 阶段六：视觉主题生成（可选）

> LLM 生成项目级 VisualTheme，一次生成作用全部幻灯片。在编排（阶段五）之前执行。

## 触发

在 `_compose_render_worker()` 中，素材绑定后、编排前：

```python
# api/routers/outlines.py
existing_theme = await get_latest_theme(project_id, db)
if not existing_theme:
    theme_input = VisualThemeInput(...)
    await generate_visual_theme(theme_input, db)
theme = await get_latest_theme(project_id, db)
# → theme 传入 compose_all_slides()
```

如已存在 draft 主题，则跳过生成，直接使用最新版本。

## 执行流程

```
1. 构建 VisualThemeInput（从 ProjectBrief + Asset 统计）
2. _load_system_prompt()  ← prompts/visual_theme_system.md
3. _build_user_message(inp)
4. call_llm_structured(system, user, VisualTheme)
5. 字号护栏检查（base_size 20-28, scale_ratio 1.2-1.5）
6. 计算 version（既有主题 +1）
7. 存入 visual_themes 表
```

## 如何收集输入

```python
# 从参考案例 Asset 中统计审美偏好
case_assets = db.query(Asset).filter(
    Asset.project_id == project_id,
    Asset.asset_type == AssetType.CASE_CARD
).all()
dominant_styles = extract_style_tags(case_assets)

# 构建 VisualThemeInput
inp = VisualThemeInput(
    project_id      = project_id,
    project_name    = brief.client_name,
    building_type   = brief.building_type,
    style_preferences = brief.style_preferences,
    dominant_styles = dominant_styles,
    narrative_hint  = brief_doc.outline_json.get("narrative_arc", ""),
)
```

## VisualTheme 如何被消费

```
VisualTheme
 ├── generate_theme_css()         → CSS 自定义属性（render/engine.py）
 └── <visual_theme> in user msg  → ComposerAgent 视觉决策上下文
```

→ 详见 [[schemas/VisualTheme]]

## 默认主题（无素材/跳过时）

如 VisualTheme 不存在，`ComposerAgent` 使用 `_default_theme()` 硬编码默认值：
- 思源黑体 + 思源宋体
- `#1C3A5F` 主色
- base_size = 22, scale_ratio = 1.333
