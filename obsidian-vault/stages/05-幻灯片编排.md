# 阶段五：幻灯片内容编排

> LLM × N 页并发调用。将每页大纲条目扩展为完整的 LayoutSpec（布局规格 + 内容块绑定）。

## 触发

在 `_compose_render_worker()` 中，素材绑定完成后自动执行：

```python
# api/routers/outlines.py line 46
async def _compose_render_worker(project_id, outline_id, db):
    await bind_outline_slides(...)        # 阶段四
    theme = await get_latest_theme(...)   # 获取 VisualTheme
    slides = await compose_all_slides(    # 本阶段
        project_id, outline_id, db,
        mode=ComposerMode.STRUCTURED
    )
    await render_and_screenshot(slides)   # 阶段七
```

## 执行流程

```
1. 加载 Outline → OutlineSpec（所有页面条目）
2. 加载 VisualTheme（最新 draft）
3. 加载所有 Asset（构建资产摘要字典）
4. 加载所有 SlideMaterialBinding（按 slide_no 索引）
5. 并发调用（asyncio.gather）：
   for entry in outline_spec.slides:
     tasks.append(_compose_slide_structured(entry, theme, brief, assets, binding))
6. 处理结果：
   - 成功 → Slide.spec_json = LayoutSpec JSON
   - 失败 → _fallback_layout_spec() → single-column 兜底
7. batch_save(slides, db)
```

## 输出路径

### v2 结构化模式（默认）

```
_ComposerLLMOutput
  → 解析 primitive_type + primitive_params
  → 构建 LayoutPrimitive 对象
  → 映射 region_bindings → RegionBinding + ContentBlock
  → 注入资产引用（"asset:uuid" → Asset.image_url 查表）
  → 保存为 Slide.spec_json
```

### v3 HTML 模式（备用）

```
_ComposerHTMLOutput.body_html
  → Slide.spec_json = {"html_mode": true, "body_html": "..."}
```

## 容错机制

| 情况 | 处理 |
|------|------|
| LLM 返回无效 JSON | `_fallback_layout_spec()` → single-column |
| primitive_type 未知 | 降级为 single-column |
| 资产 UUID 不存在 | 跳过该 block，保留其他内容 |
| LLM 超时 | `call_llm_with_limit` 截断，仍尝试解析 |

## 产出

N 条 [[schemas/Slide]]，每条：
- `spec_json` — LayoutSpec 或 html_mode JSON
- `status = "spec_ready"`

## 状态变更

```
Slide.status   → spec_ready（逐页）
Project.status → SLIDE_PLANNING
```
