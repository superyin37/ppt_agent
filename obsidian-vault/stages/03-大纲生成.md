# 阶段三：大纲生成

> LLM 调用，基于 BriefDoc + PPT_BLUEPRINT（40 页蓝图），为每个 PageSlot 生成具体内容指令。

## 触发

紧跟阶段二，在 `_outline_worker()` 中自动执行：

```python
# api/routers/outlines.py line 33
async def _outline_worker(project_id, db):
    await generate_brief_doc(project_id, db)
    await generate_outline(project_id, db)   # ← 本阶段
```

## 执行流程

```
1. 加载 ProjectBrief + BriefDoc + MaterialPackage
2. 统计参考案例数量（CASE_CARD 类型 Asset 计数）
3. _load_system_prompt(brief, brief_doc)  ← 注入 positioning_statement + narrative_arc
4. _build_blueprint_summary(reference_count)  ← 将 PPT_BLUEPRINT 转为 XML
5. 构建 user message（含 <project_brief>, <brief_doc>, <blueprint>, <material_package>）
6. call_llm_with_limit(system, user, _OutlineLLMOutput)
7. _OutlineLLMOutput → OutlineSpec → Outline ORM
8. 覆盖率分析（每页检查素材是否匹配）→ coverage_json
```

## PPT_BLUEPRINT 结构

位于 `config/ppt_blueprint.py`，共约 40 个槽位，分为 `PageSlot` 和 `PageSlotGroup`：

| 章节 | 槽位数 | 关键 slot_id |
|------|--------|-------------|
| 封面&目录 | 2 | `cover`, `toc` |
| 背景研究 | ~10 | `policy`, `policy-impact`, `urban-planning`, `traffic`, `culture`, `economy`, `economy-summary` |
| 场地分析 | ~7 | `site-location-1~4`, `site-poi`, `site-summary` |
| 竞品分析 | ~3 | `competition-local`, `competition-benchmark`, `competition-summary` |
| 参考案例 | 2-5 | `reference-case-1~N`（PageSlotGroup） |
| 项目定位 | ~2 | `positioning`, `design-strategy` |
| 设计策略 | ~4 | `concept-intro-1~3`, `design-principle` |

## 覆盖率分析

```python
# agent/outline.py (line 147-166)
for entry in outline_spec.slides:
    for key in entry.required_input_keys:
        patterns = expand_requirement(key)    # 展开为正则
        matched = find_matching_items(all_items, patterns)
        if not matched:
            missing_patterns.append(key)
    coverage_status = "complete" | "partial" | "missing"
```

Outline(

    slot_binding_hints_json = { ... },               # 每页所需输入 + 推荐匹配范围
    deck_title       = "苏州博物馆改造策划建议书",

```

## 状态变更

```

```

```
