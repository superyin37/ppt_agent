# 阶段一：素材包摄入

> 扫描本地目录，将文件分类为 MaterialItem，派生 Asset，提取 ProjectBrief。**无 LLM 调用**。

## 触发

```
POST /projects/{project_id}/material-packages/ingest-local
→ api/routers/material_packages.py (line 20)
→ ingest_local_material_package(local_path, project_id, db)
```

## 核心函数

| 函数 | 文件 | 职责 |
|------|------|------|
| `ingest_local_material_package()` | `tool/material_pipeline.py:507` | 流程总控 |
| `_derive_assets()` | `tool/material_pipeline.py:399` | MaterialItem → Asset |
| `_extract_project_brief()` | `tool/material_pipeline.py:257` | 提取项目元信息 |

## Step 1 — 文件扫描与分类

```python
# 扫描目录，将图表变体（JSON + SVG + HTML）按 basename 分组
scan_dir(local_path)
  → group chart_bundles by basename
  → classify each file by extension + filename pattern
```

### 文件名 → logical_key 映射规则

| 文件名模式 | logical_key |
|-----------|-------------|
| `参考案例N_图片_*` | `reference.case.N.images` |
| `场地分析_红线图*` | `site.boundary.image` |
| `场地分析_交通*` | `site.traffic.chart.1` |
| `经济指标_城市GDP*` | `economy.city.chart.1` |
| `政策文件_*` | `policy.national.1` 或 `.city.1` |
| `设计建议书大纲*` | `brief.outline.text` |
| `竞品分析_*` | `competition.local.1` |

## Step 2 — 创建 MaterialItem

每个文件生成一条 [[schemas/MaterialPackage]] 中的 MaterialItem 记录：

| 字段 | 处理方式 |
|------|---------|
| `logical_key` | 从文件名正则推断 |
| `kind` | 按后缀判断：`.jpg/.png/.svg`=image，`.xlsx/.csv`=spreadsheet，`.txt/.md`=document，图表三件套=chart_bundle |
| `text_content` | 文本文件前 2000 字符 |
| `structured_data` | JSON 文件解析结果，XLSX 第一个 sheet 的 10 行预览 |
| `preview_url` | `file://` URI（图片直接用原始路径） |

## Step 3 — 资产派生（`_derive_assets()`）

→ 详见 [[schemas/MaterialPackage]] 中的派生关系表

| kind | AssetType |
|------|-----------|
| image + site.* | MAP |
| image（其他） | IMAGE |
| chart_bundle | CHART |
| spreadsheet | KPI_TABLE |
| document | TEXT_SUMMARY |
| 参考案例图片组 | CASE_CARD |

## Step 4 — 提取 ProjectBrief（`_extract_project_brief()`）

从 `logical_key = "brief.outline.text"` 的 MaterialItem 文本中正则提取：

```python
# 提取容积率
re.search(r'容积率\s*[≤≥<=≧≦]\s*(\d+\.?\d*)', text)

# 提取城市
re.search(r'(北京|上海|广州|深圳|苏州|...)', text)

# 提取建筑类型
keyword_to_type = {
    "办公": "office",
    "住宅": "residential",
    "博物馆": "museum",
    ...
}
```

→ 详见 [[schemas/ProjectBrief]]

## 产出

| 模型 | 数量 |

| `Asset` | N' 条（派生，可能少于 MaterialItem 数） |
| `ProjectBrief` | 1 条 |
